# migrate_reports_to_json

Конвертирует дневные TXT-отчёты в JSON.

С версии, в которой формат дневного отчёта переехал на JSON (`{username}_dd.mm.yyyy.json`),
старые TXT-файлы перестают читаться парсером периодного отчёта и просмотрщиком.
Этот скрипт мигрирует существующие TXT.

## Что делает

- Идёт по папке с отчётами и ищет файлы вида `{USERNAME}_dd.mm.yyyy.txt`.
- Для каждого парсит содержимое (поддерживает оба легаси-формата:
  с секцией `[Поля метрик]` и без неё; целые и дробные значения нормы).
- Записывает рядом `{USERNAME}_dd.mm.yyyy.json` по схеме v1.
- TXT по умолчанию не трогает (можно удалить вручную или флагом).

## Требования

- Python 3.11+.
- Исходники проекта на машине (нужен `bootstrap.py` и `modules/`).
- Папка с отчётами на той же машине.

## Подготовка

**Сделай бэкап папки с отчётами** перед запуском:

```powershell
copy "$env:USERPROFILE\active_time" "$env:USERPROFILE\active_time.backup_pre_json" -Recurse
```

## Запуск

Из корня проекта:

```powershell
python scripts\migrate_reports_to_json.py
```

Дефолтное поведение:
- Папка отчётов: `~/active_time/` (берётся из `LOG_DIR` пользовательского `config.py`).
- Имя пользователя: значение `USERNAME` из переменных окружения Windows.
- TXT-файлы остаются на месте; для тех дней, у которых JSON уже есть, конвертация пропускается.

## Флаги

| Флаг | Назначение |
|---|---|
| `--log-dir <path>` | Указать другую папку с отчётами. По умолчанию — `LOG_DIR` из конфига. |
| `--overwrite` | Перезаписывать уже существующие JSON. По умолчанию — пропускать. |
| `--delete-txt` | Удалять TXT после успешной конвертации каждого файла. |

## Типичные сценарии

### Первая миграция на этой машине
```powershell
python scripts\migrate_reports_to_json.py
```
Открой виджет, проверь «Дневной отчёт» и «Отчёт за период». Если всё ОК — снеси TXT (см. ниже).

### Удалить TXT после успешной миграции
JSON-ы уже есть, нужно повторно пройтись и удалить TXT:
```powershell
python scripts\migrate_reports_to_json.py --overwrite --delete-txt
```
`--overwrite` нужен, чтобы скрипт не пропустил файлы (иначе видит, что JSON уже создан, и не доходит до шага удаления TXT).

### Мигрировать отчёты из другой папки
Например, после копирования с другого компьютера:
```powershell
python scripts\migrate_reports_to_json.py --log-dir "D:\imported\active_time"
```

## Подводный камень — имя пользователя

Скрипт ищет файлы по маске `{USERNAME}_*.txt`, где `USERNAME` берётся из переменной
окружения текущей Windows-сессии. Если ты копируешь файлы с компьютера, где логин был
`aaubi`, на компьютер с логином `john`, скрипт **не найдёт** файлы — он будет искать
`john_*.txt`.

Видно это сразу — в первой строке вывода скрипт печатает «Пользователь: …».

Решения:
- Переименовать TXT-файлы под нового пользователя.
- Переопределить переменную окружения в той же сессии:
  ```powershell
  $env:USERNAME = "aaubi"; python scripts\migrate_reports_to_json.py
  ```

## Откат

JSON-файлы рядом с TXT никак не мешают друг другу — старые TXT можно просто удалить руками.
Если что-то пошло не так в JSON, удали `.json`-файлы и восстанови из бэкапа.

## Схема итогового JSON (v1)

```json
{
  "version": 1,
  "username": "aaubi",
  "date": "2026-04-27",
  "first_login": "21:25:40",
  "last_logout": "21:52:05",
  "active_seconds": 8522,
  "max_work_seconds": 30600,
  "total_work_seconds": 1585,
  "session_count": 3,
  "log": [
    "2026-04-27 21:25:40 | aaubi | MONITOR_START (запуск мониторинга)"
  ]
}
```

Поля `first_login`, `last_logout`, `total_work_seconds` могут быть `null`,
если данных за день нет.
