# user_active_time
Монитор активности пользователя Windows.
Работает без прав администратора.

**Что отслеживается:**
- `LOGON` — вход в систему
- `LOGOFF` — выход из системы
- `LOCK` — блокировка экрана (Win+L)
- `UNLOCK` — разблокировка экрана
- `IDLE` — зафиксированный период простоя ввода (мышь/клавиатура не использовались)
- `MANUAL_ADD_START` / `MANUAL_ADD_END` — диапазоны, добавленные вручную через диалог

> **Модель активного времени (схема отчёта v2).** Программа записывает не «активно/неактивно», посчитанное по таймауту, а **сырые интервалы**: периоды включённой сессии (`sessions`) и периоды простоя ввода (`idle`). Активное время — это **проекция** от них с *текущим* таймаутом неактивности, вычисляемая в момент чтения/записи. Поэтому изменение таймаута пересчитывает весь день, включая уже прошедшую его часть. Старые отчёты (v1) читаются как раньше.

**Как использовать:**

1. Запусти: `python main.py`
2. Для остановки: закрой виджет (✕) или `Ctrl+C`

**Где хранятся данные:**

Всё пишется в папку `C:\Users\<пользователь>\active_time\`:
- `userName_dd.mm.yyyy.json` — дневной отчёт (структурированный JSON: активное время, сырые интервалы `sessions`/`idle`, лог событий, норма, версия схемы)
- `state.json` — внутреннее состояние программы (не редактировать)
- `config.py` — пользовательские настройки (создаётся при первом запуске)

**Конфиг:**

Проектный `config.py` в папке исходников — это **шаблон с дефолтами**, он не модифицируется приложением. При первом запуске `bootstrap.py` копирует его в `~/active_time/config.py`. Дальше всё работает с пользовательской копией: настройки из диалога пишутся туда, дефолты в репозитории остаются чистыми.

При каждом старте `bootstrap.py` пересобирает пользовательский конфиг: проектный шаблон используется как основа (структура, комментарии, набор параметров), а значения присваиваний верхнего уровня берутся из существующей пользовательской копии. Так новые параметры, добавленные в дефолты между версиями, автоматически появляются в рабочем конфиге, а уже настроенные пользователем значения сохраняются. Побочный эффект: ручные комментарии и переменные, которых нет в дефолтах, при пересборке теряются — рассчитывайте на то, что пользовательский конфиг управляется через диалог настроек.

Подробности параметров — в разделе «Настройки» ниже.

## Виджет активности

При запуске на рабочем столе появляется виджет с текущей статистикой:
- Активное время за день
- Количество сессий
- Процент активности
- Общее рабочее время (от первого логина до текущего момента)
- Сколько осталось до конца рабочего дня
- Сколько активного времени осталось до рекомендуемой нормы (`RECOMMENDED_ACTIVITY_THRESHOLD` % от рабочего времени)

Каждая метрика красится отдельно по своей шкале (фон тела нейтральный):

- **Шкала активности** (метрики «Активное время», «Активность», «Осталось активного до нормы») — по `RECOMMENDED_ACTIVITY_THRESHOLD` / `MIN_ACTIVITY_THRESHOLD`:
  - Зелёный — `≥ RECOMMENDED_ACTIVITY_THRESHOLD` (по умолчанию 80%)
  - Жёлтый — между порогами
  - Красный — `< MIN_ACTIVITY_THRESHOLD` (по умолчанию 70%)
- **Шкала рабочего времени** (метрики «Рабочее время», «Осталось до конца дня») — по `RECOMMENDED_WORK_TIME_THRESHOLD` / `MIN_WORK_TIME_THRESHOLD` (отношение `full_day_time / max_work × 100`):
  - Зелёный — норма выработана (по умолчанию ≥ 100%)
  - Жёлтый — близко к норме
  - Красный — заметно ниже нормы (по умолчанию < 80%)
- **«Количество сессий»** — без цветовой подсветки.

В заголовке виджета может отображаться текущий процент активности или счётчик «осталось до конца дня» (выбирается в настройках), а также обратный отсчёт до перехода в статус неактивности (обновляется каждую секунду). Если `INPUT_ACTIVITY_TIMEOUT = 0`, счётчик не отображается. При приближении к нулю (порог `COUNTDOWN_WARNING_SECONDS`) счётчик начинает мигать, при достижении нуля становится красным.

В нерабочие дни (рабочие часы = 0) виджет отображает сообщение «Сегодня не рабочий день» с серым фоном; метрики и счётчик скрываются.

Виджет можно перетаскивать за заголовок, сворачивать кнопкой (—) и закрывать (✕). Позиция виджета сохраняется между запусками. Закрытие виджета завершает приложение.

При достижении рекомендуемого порога активности может проигрываться звуковое уведомление (`SOUND_NOTIFICATION`).

Промежуточное сохранение данных сессии выполняется каждые `CHECKPOINT_INTERVAL` секунд, что защищает от потери данных при нештатном завершении. Открытая (ещё не закрытая) сессия персистится в `state.json` на каждом checkpoint, а при следующем запуске незакрытые из-за сбоя (например, отключение питания) сессии автоматически дотягиваются — в лог дня добавляется событие `SESSION_RECOVERED`. При жёстком сбое теряется не более одного интервала checkpoint.

## Панель инструментов

Под заголовком виджета — горизонтальная панель:

- ⏱ — добавить активное время вручную
- *(разделитель)*
- **Отчёты** ▾ — выпадающее меню:
  - Папка с отчётами (открывает `~/active_time/` в проводнике)
  - Дневной отчёт (визуализация одного дня)
  - Тепловая карта (месячный календарь с цветной заливкой по % активности)
  - Отчёт за период (выбор диапазона дат)
- ⚙ — настройки

Панель скрывается вместе с телом виджета при сворачивании.

## Добавление активного времени вручную

Кнопка ⏱ открывает модалку с тремя блоками:

- **Дата** — поле в формате `дд.мм.гггг` (по умолчанию сегодня). Можно ввести вручную или выбрать через 📅 (открывается календарь с навигацией по месяцам). При смене даты список диапазонов автоматически подтягивается за выбранный день, а добавление/удаление применяется именно к нему.
- **Добавить диапазон** — спинбоксы «С/По» (часы + минуты) и поле описания. Кнопка «Подтвердить» активна только при валидном диапазоне (`По > С`). После подтверждения форма очищается, диалог остаётся открытым — можно добавлять следующий.
- **Добавленные диапазоны** — список ранее добавленных за выбранную дату записей с чекбоксами. Кнопка «Удалить» активна, если выбран хотя бы один.

Все добавленные диапазоны попадают в дневной отчёт как пары событий `MANUAL_ADD_START`/`MANUAL_ADD_END` и учитываются в активном времени.

Для сегодняшнего (и будущих) дня правки идут в `state.json`. Прошедшие дни там не хранятся (их вычищает `cleanup_old_days` после записи отчёта), поэтому для них источником истины служит сам JSON-отчёт: добавление/удаление/список диапазонов на прошлую дату читаются и пишутся прямо в отчёт. Можно добавить время даже на дату, за которую активности вообще не было, — отчёт будет создан.

## Дневной отчёт

Меню «Отчёты → Дневной отчёт» открывает диалог выбора JSON-файла из `~/active_time/`. После выбора показывается окно со:
- Сводной статистикой за день (пользователь, дата, активное время, сессии, процент активности, начало/конец рабочего дня).
- Столбчатым графиком активности/простоя/ручного времени по часам:
  - Зелёный — периоды активности
  - Красный — периоды простоя
  - Синий — добавленное вручную время
  - Серый фон — время вне рабочего периода

Для отчётов v2 график строится из точных интервалов `sessions`/`idle` с *текущим* таймаутом неактивности (поменяете таймаут — изменится и разбивка активность/простой). Старые отчёты v1 рисуются по событиям лога, как раньше.

Если выбран не JSON-отчёт — показывается ошибка.

## Тепловая карта

Меню «Отчёты → Тепловая карта» открывает окно с месячным календарём, где каждая ячейка — день, окрашенный по проценту активности:

- Зелёный — ≥ `RECOMMENDED_ACTIVITY_THRESHOLD`% (норма достигнута)
- Жёлтый — между `MIN_ACTIVITY_THRESHOLD`% и `RECOMMENDED_ACTIVITY_THRESHOLD`%
- Красный — ниже `MIN_ACTIVITY_THRESHOLD`%
- Серый — нет данных за этот день

Сегодняшний день выделен рамкой. Дни смежных месяцев показаны приглушённо для контекста. Навигация по месяцам — кнопками `‹` `›` в шапке.

При наведении на ячейку показывается тултип: дата, активное время, норма, процент. Источник данных — те же дневные JSON-отчёты, никакой дополнительной агрегации не нужно.

## Отчёт за период

Меню «Отчёты → Отчёт за период» открывает диалог выбора дат:
- Поля «С» и «По» в формате `дд.мм.гггг`. Можно ввести вручную или выбрать через 📅 (открывается календарь с навигацией по месяцам).
- Кнопка «Построить отчёт» активна только при валидной паре дат (обе заполнены, не совпадают, «По» позже «С»).

Дни без отчётов внутри периода пропускаются. Если в выбранном диапазоне нет ни одного отчёта — показывается ошибка.

В окне отчёта:
- **Итого за период:** общее активное время и общее время работы (в часах + процент от нормы), норма, недобор активности до рекомендуемой нормы (80% от нормы) и недоработка до полной нормы (100%).
- **По дням:** таблица с детализацией по каждому дню, у которого есть отчёт. Столбцы: дата, активное (время + %), работа (время + %), норма. Строки подсвечены по дневному % активности тем же палитром, что и виджет (зелёный/жёлтый/красный/серый).

## Настройки из виджета

Кнопка ⚙ открывает диалог настроек с двумя вкладками:

**Общие:**
- *Рабочие часы* — рабочие часы по дням недели (дробные значения с шагом 0.25)
- *Оформление* — выбор темы интерфейса: «Тёмная» / «Светлая» (см. раздел [«Тема оформления»](#тема-оформления))
- *Уведомления* — звук при достижении нормы
- *Трекеры* — параметры отслеживания. Сейчас один: «Считать движение мыши за активность» (`TRACK_MOUSE_MOVE`). Применяется на лету: следующее же событие мыши после сохранения настроек уже работает по новому правилу — перезапускать приложение не нужно.
- *Таймеры* — таймаут неактивности и порог предупреждения (секунды)

**Метрики:**
- Переключатели отображения метрик в теле виджета
- Радио-кнопка выбора единственной метрики для заголовка (или «Не отображать»)

«OK» сохраняет настройки в `~/active_time/config.py` и применяет их без перезапуска (виджет, метрики, трекеры). «Отмена» закрывает окно без сохранения.

> **Замечание:** изменение значения `INPUT_ACTIVITY_TIMEOUT` применяется на лету — активное время за день (в т.ч. уже прошедшую часть) и график дневного отчёта пересчитываются под новый таймаут без перезапуска. Перезапуск нужен только при переключении мониторинга между «включено» и «выключено» (значение `0` ↔ ненулевое), т.к. от этого зависит установка хуков ввода при старте.

## Тема оформления

Интерфейс поддерживает две темы: **тёмную** (по умолчанию) и **светлую**. Тема выбирается в настройках (вкладка «Общие» → «Оформление») и сохраняется в `~/active_time/config.py` параметром `THEME` (`"dark"` / `"light"`).

Переключение применяется **на лету, без перезапуска**: виджет на рабочем столе перекрашивается сразу, а окна отчётов, тепловой карты и диалогов открываются уже в выбранной теме.

Все цветовые параметры обеих палитр вынесены в отдельный модуль `modules/theme.py`. Палитры описывают одни и те же роли (фон, текст, статусные зелёный/жёлтый/красный, подсказки и т.д.), поэтому добавить новую тему или поправить цвет можно в одном месте, не трогая UI-модули. Модули читают цвета динамически как `theme.COLOR_*` — это и обеспечивает применение темы без перезапуска.

## Мониторинг ввода (мышь/клавиатура)

Помимо отслеживания сессий, программа отслеживает активность ввода (движение мыши, клики, прокрутка, нажатия клавиш) через low-level хуки Windows.

Вместо «активно/неактивно по таймауту» фиксируются **сырые периоды простоя** — гэпы, когда ввода не было дольше `MIN_IDLE_GAP_SECONDS` (по умолчанию 5 с). Эти гэпы (`idle`) и интервалы сессий (`sessions`) — источник истины; активное время вычисляется из них с *текущим* таймаутом по формуле: для каждого гэпа «неактивная часть» = `[начало + INPUT_ACTIVITY_TIMEOUT, конец]`, а активное за день = суммарная длительность сессий минус суммарная неактивность (пересечения обрезаются границами суток).

- Простой регистрируется как событие `IDLE` и сырой интервал в `idle`
- Таймаут (`INPUT_ACTIVITY_TIMEOUT`) применяется не при записи, а при подсчёте — поэтому его изменение пересчитывает уже прошедшее время дня
- Пересчёт точен для таймаутов ≥ `MIN_IDLE_GAP_SECONDS`
- При блокировке экрана ввод игнорируется, открытый гэп финализируется
- Хуки ввода остаются «лёгкими»: фиксируется только сырой ввод, без логирования каждого события
- Не влияет на подсчёт количества сессий
- В заголовке виджета остаётся живой обратный отсчёт до перехода в простой (по текущему таймауту)

Для отключения установите `INPUT_ACTIVITY_TIMEOUT = 0` в пользовательском `config.py`.

## Настройки

Файл `~/active_time/config.py` (редактируется через диалог настроек или вручную):

- `DEFAULT_WORK_HOURS` — рабочее время по умолчанию в часах (8)
- `WORK_HOURS_BY_DAY` — рабочие часы по дням недели (`None` = `DEFAULT_WORK_HOURS`, `0` = нерабочий день). Дробные значения поддерживаются
- `RECOMMENDED_ACTIVITY_THRESHOLD` — рекомендуемый порог активности, % (80)
- `MIN_ACTIVITY_THRESHOLD` — минимально допустимый порог активности, % (70)
- `RECOMMENDED_WORK_TIME_THRESHOLD` — рекомендуемый порог рабочего времени (% от нормы) для подсветки метрик «Рабочее время» / «Осталось до конца дня». По умолчанию 100
- `MIN_WORK_TIME_THRESHOLD` — минимально допустимый порог рабочего времени, % от нормы (по умолчанию 80)
- `WIDGET_SHOW_ACTIVE_TIME` — показывать активное время в виджете (True/False)
- `WIDGET_SHOW_SESSION_COUNT` — показывать количество сессий
- `WIDGET_SHOW_ACTIVITY_PERCENT` — показывать процент активности
- `WIDGET_SHOW_FULL_DAY_TIME` — показывать общее рабочее время
- `WIDGET_SHOW_REMAINING_TIME` — показывать «осталось до конца дня»
- `WIDGET_SHOW_RECOMMENDED_REMAINING` — показывать «до рекомендуемой нормы»
- `WIDGET_SHOW_TITLE_PERCENT` — показывать процент активности в заголовке
- `WIDGET_SHOW_TITLE_REMAINING_TIME` — показывать «осталось до конца дня» в заголовке (взаимоисключаемо с другими title-флагами)
- `WIDGET_SHOW_TITLE_RECOMMENDED_REMAINING` — показывать «до рекомендуемой нормы» в заголовке (взаимоисключаемо с другими title-флагами)
- `WIDGET_UPDATE_INTERVAL` — интервал обновления данных виджета, секунды (60)
- `INPUT_ACTIVITY_TIMEOUT` — таймаут неактивности ввода, секунды (300, 0 = отключено)
- `TRACK_MOUSE_MOVE` — считать движение мыши за активность (False). Клики и скролл — всегда
- `COUNTDOWN_WARNING_SECONDS` — порог предупреждения о скором переходе в неактивность, секунды (60, 0 = отключено)
- `SOUND_NOTIFICATION` — звуковое уведомление при достижении рекомендуемого порога активности (False)
- `CHECKPOINT_INTERVAL` — интервал промежуточного сохранения сессии, секунды (60, 0 = отключено)
- `THEME` — тема оформления интерфейса: `"dark"` (тёмная) или `"light"` (светлая). По умолчанию `"dark"`. Палитры заданы в `modules/theme.py`

Если все опции отображения метрик в теле виджета выключены, тело пустое (но виджет всё равно создаётся, если `WIDGET_SHOW_TITLE_*` не выключены).

## Автозапуск без админки

Добавь ярлык в автозагрузку пользователя:
```
Win+R → shell:startup → создай ярлык на pythonw.exe main.py
```

## Миграция со старого TXT-формата

Если у тебя сохранились дневные отчёты в старом TXT-формате (`*.txt`), используй скрипт `scripts/migrate_reports_to_json.py` — он сконвертирует их в JSON. Подробности — в [scripts/migrate_reports_to_json.md](scripts/migrate_reports_to_json.md).

## Структура проекта

- `main.py` — точка входа
- `bootstrap.py` — копирует дефолтный `config.py` в `~/active_time/` при первом запуске и подключает пользовательскую копию через `sys.path`
- `config.py` — **дефолтные настройки** (сам не модифицируется приложением)
- `constants.py` — константы Windows API + шрифты, тексты UI
- `modules/theme.py` — палитры тем оформления (тёмная/светлая) и переключение темы
- `utility.py` — утилиты форматирования дат/времени
- `modules/session_monitor.py` — мониторинг сессий Windows, хранение состояния, пересчёт активного времени из сырых интервалов
- `modules/events_monitor.py` — мониторинг ввода (мышь/клавиатура) через low-level хуки; захват сырых гэпов простоя
- `modules/activity_intervals.py` — чистое ядро вычисления активного времени из интервалов `sessions`/`idle` (без I/O)
- `modules/report.py` — формирование дневного JSON-отчёта
- `modules/period_report.py` — агрегация отчёта за период из дневных JSON
- `modules/widget.py` — виджет активности на рабочем столе
- `modules/toolbar.py` — панель инструментов виджета
- `modules/manual_activity_dialog.py` — диалог ручного добавления/удаления диапазонов
- `modules/calendar_popup.py` — модальный календарь-пикер
- `modules/period_report_dialog.py` — диалог выбора периода
- `modules/period_report_viewer.py` — окно отчёта за период
- `modules/heatmap_viewer.py` — окно тепловой карты активности
- `modules/report_viewer.py` — визуализация дневного отчёта (статистика + график)
- `modules/settings_dialog.py` — диалог настроек приложения
- `modules/ui_utils.py` — общие UI-хелперы (центрирование окон)
- `scripts/migrate_reports_to_json.py` — миграция TXT-отчётов в JSON
- `tests/test_activity_intervals.py` — тесты ядра вычисления активного времени
