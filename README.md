# user_active_time
Монитор активности пользователя Windows.
Работает без прав администратора.

**Что отслеживается:**
- `LOGON` — вход в систему
- `LOGOFF` — выход из системы
- `LOCK` — блокировка экрана (Win+L)
- `UNLOCK` — разблокировка экрана
- `INPUT_INACTIVE` — пользователь неактивен (нет ввода в течение таймаута)
- `INPUT_ACTIVE` — пользователь снова активен (ввод после неактивности)
- `MANUAL_ADD_START` / `MANUAL_ADD_END` — диапазоны, добавленные вручную через диалог

**Как использовать:**

1. Запусти: `python main.py`
2. Для остановки: закрой виджет (✕) или `Ctrl+C`

**Где хранятся данные:**

Всё пишется в папку `C:\Users\<пользователь>\active_time\`:
- `userName_dd.mm.yyyy.json` — дневной отчёт (структурированный JSON: активное время, сессии, лог событий, норма)
- `state.json` — внутреннее состояние программы (не редактировать)
- `config.py` — пользовательские настройки (создаётся при первом запуске)

**Конфиг:**

Проектный `config.py` в папке исходников — это **только дефолты**, он не модифицируется приложением. При первом запуске `bootstrap.py` копирует его в `~/active_time/config.py`. Дальше всё работает с пользовательской копией: настройки из диалога пишутся туда, дефолты в репозитории остаются чистыми.

Подробности параметров — в разделе «Настройки» ниже.

## Виджет активности

При запуске на рабочем столе появляется виджет с текущей статистикой:
- Активное время за день
- Количество сессий
- Процент активности
- Общее рабочее время (от первого логина до текущего момента)
- Сколько осталось до конца рабочего дня

Цвет фона виджета меняется в зависимости от процента активности:
- Зелёный — выше рекомендуемого порога (по умолчанию 80%)
- Жёлтый — между минимальным и рекомендуемым порогом
- Красный — ниже минимального порога (по умолчанию 70%)

В заголовке виджета может отображаться текущий процент активности или счётчик «осталось до конца дня» (выбирается в настройках), а также обратный отсчёт до перехода в статус неактивности (обновляется каждую секунду). Если `INPUT_ACTIVITY_TIMEOUT = 0`, счётчик не отображается. При приближении к нулю (порог `COUNTDOWN_WARNING_SECONDS`) счётчик начинает мигать, при достижении нуля становится красным.

В нерабочие дни (рабочие часы = 0) виджет отображает сообщение «Сегодня не рабочий день» с серым фоном; метрики и счётчик скрываются.

Виджет можно перетаскивать за заголовок, сворачивать кнопкой (—) и закрывать (✕). Позиция виджета сохраняется между запусками. Закрытие виджета завершает приложение.

При достижении рекомендуемого порога активности может проигрываться звуковое уведомление (`SOUND_NOTIFICATION`).

Промежуточное сохранение данных сессии выполняется каждые `CHECKPOINT_INTERVAL` секунд, что защищает от потери данных при нештатном завершении.

## Панель инструментов

Под заголовком виджета — горизонтальная панель:

- ⏱ — добавить активное время вручную
- *(разделитель)*
- **Отчёты** ▾ — выпадающее меню:
  - Папка с отчётами (открывает `~/active_time/` в проводнике)
  - Дневной отчёт (визуализация одного дня)
  - Отчёт за период (выбор диапазона дат)
- ⚙ — настройки

Панель скрывается вместе с телом виджета при сворачивании.

## Добавление активного времени вручную

Кнопка ⏱ открывает модалку с двумя блоками:

- **Добавить диапазон** — спинбоксы «С/По» (часы + минуты) и поле описания. Кнопка «Подтвердить» активна только при валидном диапазоне (`По > С`). После подтверждения форма очищается, диалог остаётся открытым — можно добавлять следующий.
- **Добавленные диапазоны** — список ранее добавленных за сегодня записей с чекбоксами. Кнопка «Удалить» активна, если выбран хотя бы один.

Все добавленные диапазоны попадают в дневной отчёт как пары событий `MANUAL_ADD_START`/`MANUAL_ADD_END` и учитываются в активном времени.

## Дневной отчёт

Меню «Отчёты → Дневной отчёт» открывает диалог выбора JSON-файла из `~/active_time/`. После выбора показывается окно со:
- Сводной статистикой за день (пользователь, дата, активное время, сессии, процент активности, начало/конец рабочего дня).
- Столбчатым графиком активности/простоя/ручного времени по часам:
  - Зелёный — периоды активности
  - Красный — периоды простоя
  - Синий — добавленное вручную время
  - Серый фон — время вне рабочего периода

Если выбран не JSON-отчёт — показывается ошибка.

## Отчёт за период

Меню «Отчёты → Отчёт за период» открывает диалог выбора дат:
- Поля «С» и «По» в формате `дд.мм.гггг`. Можно ввести вручную или выбрать через 📅 (открывается календарь с навигацией по месяцам).
- Кнопка «Построить отчёт» активна только при валидной паре дат (обе заполнены, не совпадают, «По» позже «С»).

Если дневного отчёта нет для одной из граничных дат — показывается ошибка с указанием каких именно. Дни без отчётов внутри периода просто пропускаются.

В окне отчёта:
- **Итого за период:** общее активное время и общее время работы (в часах + процент от нормы), норма, недобор активности до рекомендуемой нормы (80% от нормы) и недоработка до полной нормы (100%).
- **По дням:** таблица с детализацией по каждому дню, у которого есть отчёт. Столбцы: дата, активное (время + %), работа (время + %), норма. Строки подсвечены по дневному % активности тем же палитром, что и виджет (зелёный/жёлтый/красный/серый).

## Настройки из виджета

Кнопка ⚙ открывает диалог настроек с двумя вкладками:

**Общие:**
- Рабочие часы по дням недели (дробные значения с шагом 0.25)
- Звук при достижении нормы
- Таймаут неактивности (секунды)
- Порог предупреждения до неактивности (секунды)

**Метрики:**
- Переключатели отображения метрик в теле виджета
- Радио-кнопка выбора единственной метрики для заголовка (или «Не отображать»)

«OK» сохраняет настройки в `~/active_time/config.py` и применяет их к виджету без перезапуска. «Отмена» закрывает окно без сохранения.

## Мониторинг ввода (мышь/клавиатура)

Помимо отслеживания сессий, программа отслеживает активность ввода (движение мыши, клики, прокрутка, нажатия клавиш). Если пользователь не взаимодействует с компьютером в течение `INPUT_ACTIVITY_TIMEOUT`, он считается неактивным.

- При отсутствии ввода — лог `INPUT_INACTIVE` (с указанием последнего источника)
- При возобновлении — лог `INPUT_ACTIVE` (с указанием источника: мышь/клавиатура)
- При блокировке экрана ввод игнорируется
- В начале каждой сессии отсчёт таймаута начинается заново
- Логируются только переходы состояния, а не каждое событие ввода
- Не влияет на подсчёт количества сессий

Для отключения установите `INPUT_ACTIVITY_TIMEOUT = 0` в пользовательском `config.py`.

## Настройки

Файл `~/active_time/config.py` (редактируется через диалог настроек или вручную):

- `DEFAULT_WORK_HOURS` — рабочее время по умолчанию в часах (8)
- `WORK_HOURS_BY_DAY` — рабочие часы по дням недели (`None` = `DEFAULT_WORK_HOURS`, `0` = нерабочий день). Дробные значения поддерживаются
- `RECOMMENDED_ACTIVITY_THRESHOLD` — рекомендуемый порог активности, % (80)
- `MIN_ACTIVITY_THRESHOLD` — минимально допустимый порог, % (70)
- `WIDGET_SHOW_ACTIVE_TIME` — показывать активное время в виджете (True/False)
- `WIDGET_SHOW_SESSION_COUNT` — показывать количество сессий
- `WIDGET_SHOW_ACTIVITY_PERCENT` — показывать процент активности
- `WIDGET_SHOW_FULL_DAY_TIME` — показывать общее рабочее время
- `WIDGET_SHOW_REMAINING_TIME` — показывать «осталось до конца дня»
- `WIDGET_SHOW_TITLE_PERCENT` — показывать процент активности в заголовке
- `WIDGET_SHOW_TITLE_REMAINING_TIME` — показывать «осталось до конца дня» в заголовке (взаимоисключаемо с предыдущим)
- `WIDGET_UPDATE_INTERVAL` — интервал обновления данных виджета, секунды (60)
- `INPUT_ACTIVITY_TIMEOUT` — таймаут неактивности ввода, секунды (300, 0 = отключено)
- `TRACK_MOUSE_MOVE` — считать движение мыши за активность (False). Клики и скролл — всегда
- `COUNTDOWN_WARNING_SECONDS` — порог предупреждения о скором переходе в неактивность, секунды (60, 0 = отключено)
- `SOUND_NOTIFICATION` — звуковое уведомление при достижении рекомендуемого порога активности (False)
- `CHECKPOINT_INTERVAL` — интервал промежуточного сохранения сессии, секунды (60, 0 = отключено)

Если все опции отображения метрик в теле виджета выключены, тело пустое (но виджет всё равно создаётся, если `WIDGET_SHOW_TITLE_*` не выключены).

## Автозапуск без админки

Добавь ярлык в автозагрузку пользователя:
```
Win+R → shell:startup → создай ярлык на pythonw.exe main.py
```

## Миграция со старого TXT-формата

Если у тебя сохранились дневные отчёты в старом TXT-формате (`*.txt`), используй скрипт `scripts/migrate_reports_to_json.py` — он сконвертирует их в JSON. Подробности — в [scripts/migrate_reports_to_json.md](scripts/migrate_reports_to_json.md).

## Структура проекта

- `main.py` — точка входа
- `bootstrap.py` — копирует дефолтный `config.py` в `~/active_time/` при первом запуске и подключает пользовательскую копию через `sys.path`
- `config.py` — **дефолтные настройки** (сам не модифицируется приложением)
- `constants.py` — константы Windows API + цвета, шрифты, тексты UI
- `utility.py` — утилиты форматирования дат/времени
- `modules/session_monitor.py` — мониторинг сессий Windows + хранение состояния
- `modules/events_monitor.py` — мониторинг ввода (мышь/клавиатура) через low-level хуки
- `modules/report.py` — формирование дневного JSON-отчёта
- `modules/period_report.py` — агрегация отчёта за период из дневных JSON
- `modules/widget.py` — виджет активности на рабочем столе
- `modules/toolbar.py` — панель инструментов виджета
- `modules/manual_activity_dialog.py` — диалог ручного добавления/удаления диапазонов
- `modules/calendar_popup.py` — модальный календарь-пикер
- `modules/period_report_dialog.py` — диалог выбора периода
- `modules/period_report_viewer.py` — окно отчёта за период
- `modules/report_viewer.py` — визуализация дневного отчёта (статистика + график)
- `modules/settings_dialog.py` — диалог настроек приложения
- `modules/ui_utils.py` — общие UI-хелперы (центрирование окон)
- `scripts/migrate_reports_to_json.py` — миграция TXT-отчётов в JSON
