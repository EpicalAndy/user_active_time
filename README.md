# user_active_time
Монитор активности пользователя Windows.
Работает без прав администратора.

**Что отслеживается:**
- `LOGON` — вход в систему
- `LOGOFF` — выход из системы
- `LOCK` — блокировка экрана (Win+L)
- `UNLOCK` — разблокировка экрана
- `INPUT_INACTIVE` — пользователь неактивен (нет ввода в течение таймаута)
- `INPUT_ACTIVE` — пользователь снова активен (ввод после неактивности)

**Как использовать:**

1. Запусти: `python main.py`
2. Для остановки: закрой виджет (✕) или `Ctrl+C`

**Где хранятся данные:**

Всё пишется в папку `C:\Users\<пользователь>\active_time\`:
- `userName_dd.mm.yyyy.txt` — отчёт за день (активное время, сессии, лог событий)
- `state.json` — внутреннее состояние программы (не редактировать)

**Виджет активности:**

При запуске на рабочем столе появляется виджет с текущей статистикой:
- Активное время за день
- Количество сессий
- Процент активности
- Общее рабочее время (от первого логина до текущего момента)

Цвет фона виджета меняется в зависимости от процента активности:
- Зелёный — выше рекомендуемого порога (по умолчанию 80%)
- Жёлтый — между минимальным и рекомендуемым порогом
- Красный — ниже минимального порога (по умолчанию 70%)

В заголовке виджета отображается обратный отсчёт до перехода в статус неактивности
(обновляется каждую секунду). Если `INPUT_ACTIVITY_TIMEOUT = 0`, счётчик не отображается.
При приближении к нулю (порог задаётся `COUNTDOWN_WARNING_SECONDS`) счётчик начинает мигать,
а при достижении нуля становится красным.

В нерабочие дни (рабочие часы = 0) виджет отображает сообщение «Сегодня не рабочий день»
с серым фоном, метрики и счётчик обратного отсчёта скрываются.

Виджет можно перетаскивать за заголовок, сворачивать кнопкой (—) и закрывать (✕).
Позиция виджета сохраняется между запусками.
Закрытие виджета завершает приложение.

При достижении рекомендуемого порога активности может проигрываться звуковое уведомление
(управляется параметром `SOUND_NOTIFICATION`).

Промежуточное сохранение данных сессии выполняется каждые `CHECKPOINT_INTERVAL` секунд,
что защищает от потери данных при нештатном завершении приложения.

**Панель инструментов:**

Под заголовком расположена панель инструментов с иконками (при наведении появляется подсказка):
- 📂 — открыть папку с отчётами в проводнике
- 📊 — визуализация отчёта (выбор файла → окно со статистикой и графиком активности)
- ⚙ — открыть окно настроек приложения

Панель скрывается вместе с телом виджета при сворачивании.

**Визуализация отчёта:**

По нажатию на 📊 в панели инструментов открывается диалог выбора файла отчёта.
После выбора файла отображается окно с:
- Сводная статистика за день (пользователь, дата, активное время, сессии, процент активности)
- Столбчатый график активности/простоя за день (по часам)
  - Зелёный — периоды активности
  - Красный — периоды простоя
  - Серый фон — время вне рабочего периода

При выборе файла выполняется валидация: если файл не является отчётом об активности,
выводится сообщение об ошибке.

**Настройки из виджета:**

По нажатию на ⚙ в панели инструментов открывается модальное окно настроек:
- Рабочие часы по дням недели (дробные значения с шагом 0.25)
- Переключатели отображения метрик в виджете
- Включение/выключение звукового уведомления
- Таймаут неактивности ввода (секунды)
- Порог предупреждения до неактивности (секунды)

Кнопка «OK» сохраняет изменения в `config.py` и применяет их к виджету без перезапуска.
Кнопка «Отмена» закрывает окно без сохранения.

**Мониторинг ввода (мышь/клавиатура):**

Помимо отслеживания сессий, программа отслеживает активность ввода пользователя
(движение мыши, клики, прокрутка, нажатия клавиш). Если пользователь не взаимодействует
с компьютером в течение заданного таймаута, он считается неактивным.

- При отсутствии ввода в течение таймаута — лог `INPUT_INACTIVE` (с указанием последнего источника)
- При возобновлении ввода — лог `INPUT_ACTIVE` (с указанием источника: мышь/клавиатура)
- При блокировке экрана ввод игнорируется
- В начале каждой сессии отсчёт таймаута начинается заново
- Логируются только переходы состояния, а не каждое событие ввода
- Не влияет на подсчёт количества сессий

Для отключения мониторинга ввода установите `INPUT_ACTIVITY_TIMEOUT = 0` в `config.py`.

**Настройки:**

Файл `config.py`:
- `DEFAULT_WORK_HOURS` — рабочее время по умолчанию в часах (по умолчанию 8)
- `WORK_HOURS_BY_DAY` — рабочие часы по дням недели (None = используется `DEFAULT_WORK_HOURS`, 0 = не отслеживать). Поддерживает дробные значения (например, 7.50)
- `RECOMMENDED_ACTIVITY_THRESHOLD` — рекомендуемый порог активности, % (по умолчанию 80)
- `MIN_ACTIVITY_THRESHOLD` — минимально допустимый порог, % (по умолчанию 70)
- `WIDGET_SHOW_ACTIVE_TIME` — показывать активное время в виджете (True/False)
- `WIDGET_SHOW_SESSION_COUNT` — показывать количество сессий (True/False)
- `WIDGET_SHOW_ACTIVITY_PERCENT` — показывать процент активности (True/False)
- `WIDGET_SHOW_FULL_DAY_TIME` — показывать общее рабочее время от первого логина до текущего момента (True/False)
- `WIDGET_UPDATE_INTERVAL` — интервал обновления данных виджета в секундах (по умолчанию 60)
- `INPUT_ACTIVITY_TIMEOUT` — таймаут неактивности ввода в секундах (по умолчанию 300, 0 = отключено)
- `TRACK_MOUSE_MOVE` — считать движение мыши за активность (по умолчанию False). Клики и скролл отслеживаются всегда
- `COUNTDOWN_WARNING_SECONDS` — порог предупреждения о скором переходе в неактивность, секунды (по умолчанию 60, 0 = отключено)
- `SOUND_NOTIFICATION` — звуковое уведомление при достижении рекомендуемого порога активности (по умолчанию False)
- `CHECKPOINT_INTERVAL` — интервал промежуточного сохранения сессии в секундах (по умолчанию 60, 0 = отключено)

Если все четыре опции отображения выключены (False), виджет не создаётся.

**Автозапуск без админки:**

Добавь ярлык в автозагрузку пользователя:
```
Win+R → shell:startup → создай ярлык на pythonw.exe main.py
```

**Структура проекта:**
- `main.py` — точка входа
- `config.py` — настройки приложения
- `constants.py` — константы Windows API
- `utility.py` — утилиты (форматирование дат/времени)
- `modules/report.py` — формирование отчёта
- `modules/session_monitor.py` — мониторинг сессий Windows
- `modules/events_monitor.py` — мониторинг ввода (мышь/клавиатура)
- `modules/widget.py` — виджет активности на рабочем столе
- `modules/toolbar.py` — панель инструментов виджета
- `modules/report_viewer.py` — визуализация отчёта (статистика + график)
- `modules/settings_dialog.py` — диалог настроек приложения
